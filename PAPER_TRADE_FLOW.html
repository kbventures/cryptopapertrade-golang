Paper Trade — System Flow

— opening a trade
◎
User opens a paper trade
Sets entry price, stop-loss, and take-profit levels from the mobile app.
▦
Save open trade to database
Trade record created with status OPEN, entry price, SL, and TP stored.
Postgres
⬡
Server watches price feed
Continuous loop via SSE / WebSocket from exchange (CCXT). Tracks all open trades against their SL / TP levels.
SSE / WS
— price target hit
⚡
Price hits Stop Loss or Take Profit
Condition matched — trade closure triggered instantly.
▦
Update trade record to CLOSED
Final P&L, exit price, and timestamp written. Status set to CLOSED.
Postgres
two things happen in parallel
Notify user
Push notification sent immediately — trade result delivered to mobile.
Push notif
Queue AI analysis job
Job pushed to Asynq / Redis. Server not blocked. Survives restarts.
Asynq / Redis
⬡
Background worker calls Claude API
Sends trade data (entry, exit, chart context). Takes 3–10 s. Retries on failure.
Claude API
▦
Save AI analysis to trade record
Analysis attached to the existing closed trade row in Postgres.
Postgres
◎
User views trade + AI critique
Mobile app fetches the closed trade — result and full AI breakdown available.
